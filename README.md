# メモ帳、タスク管理一括管理webアプリ

## はじめに
やらなければならないことが増えるとすぐ忘れて失敗することが多いため、
- 期限があるものをタスクとして
- 期限がないものをメモとして
扱うことが多い

→一括で管理できるアプリを作ってみたい！

と思い、java/jspとMySQLを使って簡単なwebアプリケーションを作ってみました

jspの基礎やデータベース連携(MySQL)などは[10日でおぼえるJSP&サーブレット入門教室 第4版](https://www.amazon.co.jp/10%E6%97%A5%E3%81%A7%E3%81%8A%E3%81%BC%E3%81%88%E3%82%8BJSP-%E3%82%B5%E3%83%BC%E3%83%96%E3%83%AC%E3%83%83%E3%83%88%E5%85%A5%E9%96%80%E6%95%99%E5%AE%A4-%E7%AC%AC4%E7%89%88-%E5%B1%B1%E7%94%B0-%E7%A5%A5%E5%AF%9B/dp/4798140341)を読んで学びました

## 反省・改善点

このwebアプリの作成にあたってコーディングのスキルなどはもちろんのこと、もっとこうしたいなどと思ったことを書いていきます

- 表示ページについて
  - データベースを横並びにする
  実際に使っていくとメモやタスクが多くなって縦に並びきらない（スライドしないといけない）ようになると思うので縦ではなく横に並べたいなと思いました

- メモ・タスク編集ページについて
  - 遷移先を減らしたい
  それぞれボタンから遷移して操作するのではなく、テキスト欄に入力してボタンを押すとその操作が行われるようにしたいです（=プログラムを減らしたい）
  - 検索・削除ページをもっとリアリティのあるものにしたい
  両方とも完全一致でしか検索・削除できないので部分一致であったり、削除に関してはボタンを押すだけで対象を削除できるようにしたいです
  - ソート機能の改善（タスク編集ページに限る）
  実際にタスク管理をする際に西暦何年までは入力しないため、今回のデータベースでは存在しないようにしているが、
  今回のこのソート機能は月日でソートを行うため、年を越すようなタスクが出現したときにうまくソートできなくなっています（12月より1月を優先してしまう）
  データベース内の月日を扱えるようになってソート機能の改善を行い、さらに残り何日かまで表示できるようにしたいです
  - 内容変更をできるようにしたい
  書き込みにミスが生じたり、変更したいときにこのままだと削除してからまた追加しなければいけません
  実際に使うには不便すぎるので変更機能をつけたいです。

- さらにデザインしたい
  背景やボタンなどまだまだデフォルトのモノばかりなのでよりアレンジしたいです
